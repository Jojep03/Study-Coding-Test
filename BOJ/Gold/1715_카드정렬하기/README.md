# 🥇 BOJ 1715 - 카드 정렬하기

## 📌 문제 설명

여러 개의 카드 묶음이 주어질 때,  
두 묶음을 합칠 때마다 **비교 횟수 = 두 묶음 크기의 합**이 됩니다.

모든 카드 묶음을 하나로 합칠 때,  
**최소 비교 횟수**를 구하는 문제입니다.


## 💡 핵심 아이디어

### ❗ 가장 작은 두 묶음을 먼저 합쳐야 한다

이 문제는 **그리디 + 최소 힙** 문제입니다.

카드 묶음을 합칠 때마다 합친 카드 수만큼 비교 횟수가 발생합니다.  
따라서 큰 묶음을 먼저 합치면 이후에도 계속 큰 비용이 누적됩니다.

그래서 매번 **가장 작은 두 묶음**을 먼저 합쳐야 전체 비교 횟수를 최소로 만들 수 있습니다.

예를 들어 카드 묶음이 다음과 같다고 해봅시다.

~~~txt
10 20 40
~~~

### 잘못된 방법

큰 묶음부터 합치면 다음과 같습니다.

~~~txt
40 + 20 = 60
60 + 10 = 70

총합 = 60 + 70 = 130
~~~

### 올바른 방법

가장 작은 두 묶음부터 합치면 다음과 같습니다.

~~~txt
10 + 20 = 30
30 + 40 = 70

총합 = 30 + 70 = 100
~~~

따라서 **매번 가장 작은 두 카드 묶음을 먼저 합치는 것이 최적**입니다.


## ⚙️ 사용 알고리즘

## 최소 힙

파이썬에서는 `heapq`를 사용하면 최소 힙을 쉽게 구현할 수 있습니다.

- `heapq.heappush(heap, x)`  
  - 힙에 값 `x`를 넣습니다.

- `heapq.heappop(heap)`  
  - 힙에서 가장 작은 값을 꺼냅니다.

이 문제에서는 매번 가장 작은 두 묶음을 꺼내야 하므로 최소 힙이 적합합니다.


## 🔄 풀이 과정

1. 모든 카드 묶음을 최소 힙에 저장합니다.
2. 힙에서 가장 작은 카드 묶음 두 개를 꺼냅니다.
3. 두 묶음을 합친 값을 `res`에 더합니다.
4. 합쳐진 새 카드 묶음을 다시 힙에 넣습니다.
5. 힙에 카드 묶음이 1개만 남을 때까지 반복합니다.



## 가장 작은 두 묶음 꺼내기

~~~python
first = heapq.heappop(heap)
second = heapq.heappop(heap)
~~~

현재 남아 있는 카드 묶음 중  
가장 작은 두 묶음을 꺼냅니다.

---

## 두 묶음 합치기

~~~python
total = first + second
res += total
~~~

두 카드 묶음을 합친 값이 이번 비교 횟수입니다.

따라서 `res`에 `total`을 더합니다.

---

## 합친 묶음 다시 넣기

~~~python
heapq.heappush(heap, total)
~~~

합쳐진 카드 묶음은 이후에도 다시 다른 카드 묶음과 합쳐져야 합니다.

따라서 다시 힙에 넣어줍니다.

## ⏱️ 시간 복잡도

힙에서 값을 꺼내거나 넣는 연산은 각각 `O(log N)`입니다.

각 반복마다 다음 연산을 수행합니다.

- `heappop` 2번
- `heappush` 1번

카드 묶음을 하나로 합치려면 총 `N - 1`번 반복하므로,  
전체 시간 복잡도는 다음과 같습니다.

~~~txt
O(N log N)
~~~


## 🧠 핵심 정리

~~~txt
가장 작은 두 카드 묶음을 반복해서 합친다.
합친 비용을 누적한다.
합친 묶음은 다시 힙에 넣는다.
카드 묶음이 1개 남으면 종료한다.
~~~
