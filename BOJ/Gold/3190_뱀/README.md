# 🐍 BOJ 3190 - 뱀

## 🔗 문제 링크

https://www.acmicpc.net/problem/3190


## 📌 문제 설명

N×N 보드 위에서 뱀이 이동한다.

* 처음 위치는 `(1, 1)`
* 처음 방향은 오른쪽
* 사과가 있는 칸으로 이동하면 길이가 늘어난다.
* 사과가 없으면 꼬리가 한 칸 줄어든다.
* 벽이나 자기 몸에 부딪히면 게임이 끝난다.
* 특정 시간이 되면 방향을 회전한다.

게임이 몇 초에 끝나는지 구하는 문제이다.


## 💡 핵심 아이디어

이 문제는 **시뮬레이션** 문제이다.

매 초마다 다음 과정을 반복하면 된다.

1. 현재 방향으로 머리를 한 칸 이동
2. 벽 또는 자기 몸과 충돌하는지 확인
3. 사과가 있으면 몸 길이 증가
4. 사과가 없으면 꼬리 제거
5. 해당 시간이면 방향 전환

즉,
**“머리를 늘리고, 필요하면 꼬리를 줄이는 과정”** 을 그대로 구현하면 된다.


## 🧠 사용한 자료구조

### 1. `deque`

뱀의 몸 전체를 저장한다.

* 머리 추가: `appendleft()`
* 꼬리 제거: `pop()`

뱀은 앞쪽으로 이동하고 뒤쪽이 줄어드는 구조이므로 `deque`가 잘 맞는다.

### 2. `set`

현재 뱀의 몸이 있는 좌표를 저장한다.

* `(r, c) in snake_set` 으로 몸 충돌을 빠르게 확인 가능
* `deque`만 쓰면 몸 충돌 검사에 시간이 더 걸릴 수 있으므로 `set`을 함께 사용한다.



## 🔄 방향 처리

방향을 시계 방향 순서로 관리하면 회전 처리가 쉬워진다.

```text
오른쪽 → 아래 → 왼쪽 → 위
```

```python
dr = [0, 1, 0, -1]
dc = [1, 0, -1, 0]
```

* `D` : 오른쪽 회전 → `(direction + 1) % 4`
* `L` : 왼쪽 회전 → `(direction - 1) % 4`



## ⚠️ 주의할 점

### 1. 좌표는 1부터 시작한다

입력 좌표는 `(1, 1)`부터 시작하지만, 파이썬 배열은 `0`부터 시작한다.
그래서 사과 위치를 저장할 때 반드시 `-1` 보정이 필요하다.

```python
board[r - 1][c - 1] = 1
```

### 2. 방향 전환은 이동 후에 한다

문제에서 X초가 끝난 뒤 방향을 전환하므로,
**이동을 먼저 하고 나서 방향을 바꿔야 한다.**

### 3. 꼬리 위치는 예외가 될 수 있다

사과가 없다면 이번 턴에 꼬리가 빠지므로,
머리가 원래 꼬리 위치로 들어가는 경우는 허용될 수 있다.
이 부분 때문에 몸 충돌 처리 순서를 조심해야 한다.



## ✅ 코드 중요 부분 설명

### 1. 사과 위치 저장

```python
for _ in range(k):
    r, c = map(int, input().split())
    board[r - 1][c - 1] = 1
```

* 보드에서 사과 위치를 `1`로 표시한다.
* 좌표는 1-based 입력이므로 `-1` 해줘야 한다.


### 2. 방향 전환 정보 저장

```python
turns = deque()
for _ in range(l):
    x, d = input().split()
    turns.append((int(x), d))
```

* 몇 초 뒤에 어느 방향으로 회전할지 저장한다.
* 앞에서부터 순서대로 확인하면 되므로 `deque`를 사용했다.


### 3. 뱀의 몸 초기화

```python
snake = deque([(0, 0)])
snake_set = set([(0, 0)])
```

* 처음 뱀은 `(0, 0)` 한 칸만 차지하고 있다.
* `snake`는 몸 순서를 관리하고,
* `snake_set`은 몸 충돌을 빠르게 검사하기 위해 사용한다.


### 4. 다음 머리 위치 계산

```python
head_r, head_c = snake[0]
nr = head_r + dr[direction]
nc = head_c + dc[direction]
```

* 현재 머리 위치에서
* 현재 방향만큼 이동한 다음 위치를 계산한다.


### 5. 벽 충돌 검사

```python
if nr < 0 or nr >= n or nc < 0 or nc >= n:
    break
```

* 보드 범위를 벗어나면 게임 종료

### 6. 몸 충돌 검사

```python
tail_r, tail_c = snake[-1]

if (nr, nc) in snake_set and (nr, nc) != (tail_r, tail_c):
    break
```

* 다음 위치가 현재 뱀 몸에 포함되어 있으면 충돌이다.
* 하지만 꼬리는 이번 턴에 빠질 수도 있기 때문에,
  **다음 위치가 현재 꼬리 위치인 경우는 예외로 처리**한다.


### 7. 머리 이동

```python
snake.appendleft((nr, nc))
snake_set.add((nr, nc))
```

* 새 머리를 `deque`의 앞쪽에 추가한다.
* 동시에 `set`에도 넣어서 현재 몸 좌표를 갱신한다.


### 8. 사과가 있는지 확인

```python
if board[nr][nc] == 1:
    board[nr][nc] = 0
else:
    tr, tc = snake.pop()
    snake_set.remove((tr, tc))
```

* 사과가 있으면 꼬리를 그대로 둬서 몸 길이가 늘어난다.
* 사과가 없으면 꼬리를 제거해서 길이를 유지한 채 앞으로 이동한다.

즉,

* **사과 있음** → 머리 추가만
* **사과 없음** → 머리 추가 + 꼬리 제거

### 9. 방향 전환

```python
if turns and turns[0][0] == time:
    _, d = turns.popleft()
    if d == 'D':
        direction = (direction + 1) % 4
    else:
        direction = (direction - 1) % 4
```

* 현재 시간이 방향 전환 시간과 같으면 회전한다.
* `D`는 오른쪽 회전, `L`은 왼쪽 회전이다.


## ❗ 오답 포인트
1. 좌표 보정 안 하면 틀림
```python
board[r - 1][c - 1]
```

2. 방향 전환 시점

    👉 이동 후 회전해야 한다


3. 꼬리 예외 처리 안 하면 틀림

    👉 꼬리는 이번 턴에 사라질 수 있음


4. deque만 쓰면 시간 비효율

    👉 set 같이 써야 충돌 체크 빠름
## ⏱️ 시간 복잡도

각 초마다 수행하는 연산은 대부분 O(1)이다.

* 머리 이동: O(1)
* 꼬리 제거: O(1)
* 몸 충돌 검사: O(1)

따라서 전체는 시뮬레이션 횟수에 비례하며,
문제 조건 안에서 충분히 빠르게 동작한다.
