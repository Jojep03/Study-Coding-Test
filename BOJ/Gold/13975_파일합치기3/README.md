# 🥇 BOJ 13975 - 파일 합치기 3

## 📌 문제 설명

여러 개의 파일이 주어질 때,  
두 파일을 합칠 때마다 **비용 = 두 파일 크기의 합**이 됩니다.

모든 파일을 하나로 합칠 때,  
**최소 비용**을 구하는 문제입니다.


## 💡 핵심 아이디어

### ❗ 가장 작은 두 파일을 먼저 합쳐야 한다

이 문제는 **그리디 + 최소 힙** 문제입니다.

파일을 합칠 때마다 합친 파일 크기만큼 비용이 발생합니다.  
따라서 큰 파일을 먼저 합치면 이후에도 계속 큰 비용이 누적됩니다.

그래서 매번 **가장 작은 두 파일**을 먼저 합쳐야 전체 비용을 최소로 만들 수 있습니다.

이 풀이 방식은 **BOJ 1715 - 카드 정렬하기**와 거의 동일합니다.

예를 들어 파일 크기가 다음과 같다고 해봅시다.

~~~txt
40 30 30 50
~~~

### 파일을 작은 순서로 합치면

~~~txt
30 + 30 = 60
40 + 50 = 90
60 + 90 = 150

총합 = 60 + 90 + 150 = 300
~~~

따라서 **매번 가장 작은 두 파일을 먼저 합치는 것이 최적**입니다.


## ⚙️ 사용 알고리즘

## 최소 힙

파이썬에서는 `heapq`를 사용하면 최소 힙을 쉽게 구현할 수 있습니다.

- `heapq.heappush(heap, x)`  
  - 힙에 값 `x`를 넣습니다.

- `heapq.heappop(heap)`  
  - 힙에서 가장 작은 값을 꺼냅니다.

이 문제에서는 매번 가장 작은 두 파일을 꺼내야 하므로 최소 힙이 적합합니다.


## 🔄 풀이 과정

1. 테스트케이스 개수 `t`를 입력받습니다.
2. 각 테스트케이스마다 파일 크기들을 최소 힙에 저장합니다.
3. 힙에서 가장 작은 파일 두 개를 꺼냅니다.
4. 두 파일을 합친 값을 `res`에 더합니다.
5. 합쳐진 새 파일을 다시 힙에 넣습니다.
6. 힙에 파일이 1개만 남을 때까지 반복합니다.


## ✅ 정답 코드

~~~python
import heapq
import sys

input = sys.stdin.readline

t = int(input())

for _ in range(t):
    k = int(input())
    files = map(int, input().split())

    heap = []
    for file in files:
        heapq.heappush(heap, file)

    res = 0

    for _ in range(k - 1):
        first = heapq.heappop(heap)
        second = heapq.heappop(heap)

        cost = first + second
        res += cost

        heapq.heappush(heap, cost)

    print(res)
~~~


## ⏱️ 시간 복잡도

힙에서 값을 꺼내거나 넣는 연산은 각각 `O(log K)`입니다.

각 반복마다 다음 연산을 수행합니다.

- `heappop` 2번
- `heappush` 1번

파일을 하나로 합치려면 총 `K - 1`번 반복하므로,  
전체 시간 복잡도는 다음과 같습니다.

~~~txt
O(K log K)
~~~


## 🧠 핵심 정리

~~~txt
가장 작은 두 파일을 반복해서 합친다.
합친 비용을 누적한다.
합친 파일은 다시 힙에 넣는다.
파일이 1개 남으면 종료한다.
~~~
