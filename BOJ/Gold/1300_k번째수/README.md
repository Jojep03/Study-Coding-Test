# 🥇 BOJ 1300 - K번째 수

## 📌 문제 링크
https://www.acmicpc.net/problem/1300

---

## 📖 문제 설명

N × N 크기의 배열 B가 다음과 같이 정의됩니다.

B[i][j] = i × j


이 배열을 1차원으로 펼친 뒤 오름차순 정렬했을 때,  
**K번째에 해당하는 수**를 구하는 문제입니다.

---

## 💡 풀이 아이디어

### ❌ 단순 접근 (실패)
- 모든 값을 배열에 넣으면 크기가 최대 \(N^2\)
- N ≤ 100,000 이므로 최대 \(10^{10}\) → 메모리 & 시간 초과

---

### ✅ 핵심 아이디어: **이분 탐색 (값 기준 탐색)**

정렬된 배열을 직접 만들지 않고  
**"K번째 수가 될 값"을 이분 탐색으로 찾는다**

---

### 🔍 핵심 로직

`mid`를 기준으로 생각합니다.

- `cnt = mid 이하인 값의 개수`

그럼 다음이 성립합니다:

- `cnt >= k`
  → **mid 이하에 이미 k개 이상 존재**
  → K번째 수는 **mid보다 작거나 같음**
  → 👉 왼쪽 탐색 (값 줄이기)

- `cnt < k`
  → **mid 이하의 개수가 부족**
  → K번째 수는 **mid보다 큼**
  → 👉 오른쪽 탐색 (값 키우기)

### 📌 예시로 이해하기

n = 3일 때 배열:

1 2 3

2 4 6

3 6 9

정렬하면:

1 2 2 3 3 4 6 6 9


#### if k = 5 → 정답은 **3**

---

#### ✔️ mid = 4

4 이하인 수:

1 2 2 3 3 4 → 총 6개

👉 `cnt = 6 ≥ 5`

→ K번째 수는 **4보다 작거나 같음**


#### ✔️ mid = 2

2 이하인 수:

1 2 2 → 총 3개


👉 `cnt = 3 < 5`

→ K번째 수는 **2보다 큼**

---

### 🎯 핵심 정리

> "mid 이하의 개수"로  
> K번째 값이 mid의 왼쪽인지 오른쪽인지 판단한다.


### 🔢 개수 세는 방법

i번째 행은 다음과 같습니다:

i × 1, i × 2, i × 3, ..., i × n

→ `mid` 이하인 값의 개수는

`mid // i`


하지만 최대 n개를 넘을 수 없으므로

`min(mid // i, n)`


## 🚀 핵심 포인트 정리
- `배열을 만들지 않는다`
- `값 자체를 이분 탐색한다`
- mid 이하 개수를 통해 정답 위치 판단
- `min(mid // i, n)`이 핵심


## ⏱ 시간 복잡도
- 이분 탐색: `O(log k)`
- 각 탐색마다 N번 반복

**👉 총 시간 복잡도: O(N log k)**
