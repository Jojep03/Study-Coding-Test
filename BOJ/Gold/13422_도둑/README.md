# 백준 13422 - 도둑

## 문제 핵심

마을의 집들이 **원형으로 배치**되어 있고, 도둑은 **연속된 `M`개의 집**에서 돈을 훔친다.

훔친 돈의 총합이 방범장치가 작동하는 기준 금액 `K`보다 작으면 도둑질에 성공한다.

즉,

```python
연속된 M개 집의 돈의 합 < K
```

를 만족하는 경우의 수를 구하면 된다.

---

## 풀이 방법

연속된 `M`개의 합을 계속 구해야 하므로 **슬라이딩 윈도우**를 사용한다.

처음 `M`개의 합을 구한다.

```python
temp = sum(homes[:m])
```

그다음 윈도우를 한 칸씩 이동시키면서

* 가장 왼쪽 집의 돈을 빼고
* 새롭게 들어오는 집의 돈을 더한다.

```python
temp -= homes[i]
temp += homes[(i + m) % n]
```

집들이 원형으로 연결되어 있기 때문에

```python
(i + m) % n
```

을 사용하면 마지막 집을 넘어갔을 때 다시 첫 번째 집으로 돌아갈 수 있다.

예를 들어 `N = 5`, `M = 3`이라면 확인하는 구간은

```text
[0, 1, 2]
[1, 2, 3]
[2, 3, 4]
[3, 4, 0]
[4, 0, 1]
```

이 된다.

---

## 주의할 점

### `N == M`인 경우

모든 집을 한 번에 훔치는 경우이므로 가능한 집의 조합은 **단 1개**뿐이다.

일반적인 원형 슬라이딩 윈도우처럼 `N`번 반복하면 같은 전체 집의 합을 `N`번 중복해서 세게 된다.

따라서 따로 처리한다.

```python
if n == m:
    if temp < k:
        cnt = 1
```

---

## 코드

```python
import sys

input = sys.stdin.readline

t = int(input())

for _ in range(t):
    n, m, k = map(int, input().split())
    homes = list(map(int, input().split()))

    temp = sum(homes[:m])
    cnt = 0

    if n == m:
        if temp < k:
            cnt = 1
    else:
        for i in range(n):
            if temp < k:
                cnt += 1

            temp -= homes[i]
            temp += homes[(i + m) % n]

    print(cnt)
```

## 핵심 정리

* **연속된 고정 길이 `M`의 합** → 슬라이딩 윈도우
* **원형 배열** → `% n`을 이용하여 처음으로 돌아가기
* 성공 조건은 `temp < k`
* `N == M`이면 가능한 경우는 하나뿐이므로 예외 처리

### 시간 복잡도

각 테스트 케이스마다 처음 `M`개의 합을 구한 뒤 최대 `N`번 이동한다.

```text
O(N)
```
