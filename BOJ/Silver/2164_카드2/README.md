# 🥇 BOJ 2164 - 카드2

## 🔗 문제 링크

https://www.acmicpc.net/problem/2164

## 📌 문제 설명

1번부터 N번까지 카드가 순서대로 놓여있다.

다음 과정을 반복한다:

1. 맨 위 카드를 버린다.
2. 그 다음 맨 위 카드를 맨 아래로 옮긴다.

카드가 한 장 남을 때, 마지막 카드 번호를 구하는 문제이다.

## 💡 풀이 아이디어

이 문제는 **큐(Deque)를 이용한 시뮬레이션** 문제이다.

* 카드의 앞/뒤 이동이 많기 때문에 `deque`를 사용한다.
* 과정을 그대로 구현하면 된다.

과정:

* 맨 앞 카드 제거 → `popleft()`
* 그 다음 카드 꺼내서 뒤로 이동 → `append(popleft())`
* 카드가 1장 남을 때까지 반복


## ⚙️ 알고리즘

1. `deque`에 1부터 N까지 넣는다.
2. 카드가 1장 남을 때까지 반복

   * 맨 앞 카드 제거
   * 다음 카드 뒤로 이동
3. 마지막 카드 출력


## ⏱ 시간 복잡도

* 각 카드가 한 번씩 처리됨 → **O(n)**
* deque 연산 → O(1)

👉 전체 시간 복잡도: **O(n)**

## 🚀 추가 풀이 (수학적 접근)

이 문제는 규칙이 존재한다.

* N이 2의 거듭제곱이면 → 정답 = N
* 아니라면 →

  ```
  정답 = 2 × (N - 가장 큰 2의 거듭제곱)
  ```

### 예시

* N = 6
  → 가장 큰 2의 거듭제곱 = 4
  → 정답 = 2 × (6 - 4) = 4


## 🧠 핵심 정리

* deque로 푸는 대표적인 큐 시뮬레이션 문제
* 시간 복잡도 O(n)으로 충분히 통과 가능
* 수학적 패턴으로도 풀 수 있는 문제
