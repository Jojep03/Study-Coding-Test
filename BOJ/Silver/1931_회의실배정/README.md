# 🥇 BOJ 1931 - 회의실 배정

## 📌 문제 설명

여러 개의 회의가 주어질 때,  
**한 개의 회의실에서 겹치지 않게 사용할 수 있는 최대 회의 수**를 구하는 문제입니다.

각 회의는 다음과 같이 주어집니다.

- 시작 시간
- 종료 시간

회의가 끝나는 시간과 다음 회의가 시작하는 시간이 같아도  
두 회의는 이어서 진행할 수 있습니다.


## 💡 핵심 아이디어

이 문제는 **그리디 알고리즘** 문제입니다.

최대한 많은 회의를 선택하려면  
**끝나는 시간이 빠른 회의부터 선택**해야 합니다.

왜냐하면 현재 회의가 빨리 끝날수록  
뒤에 더 많은 회의를 배정할 수 있기 때문입니다.

따라서 회의를 정렬할 때는 다음 기준을 사용합니다.

1. 종료 시간이 빠른 순서
2. 종료 시간이 같다면 시작 시간이 빠른 순서

```python
lst.sort(key=lambda x: (x[1], x[0]))
```

## ⚙️ 사용 알고리즘
- 그리디
- 정렬


## 🔄 풀이 과정
1. 회의 정보를 리스트에 저장합니다.

2. 종료 시간 기준으로 오름차순 정렬합니다.

3. 현재 선택한 회의의 종료 시간을 저장합니다.

4. 다음 회의의 시작 시간이 현재 종료 시간 이상이면 선택합니다.

5. 선택한 회의 개수를 출력합니다.

## ✅ 정답 코드

```python
import sys

input = sys.stdin.readline

n = int(input())
lst = []

for _ in range(n):
    st, en = map(int, input().split())
    lst.append((st, en))

lst.sort(key=lambda x: (x[1], x[0]))

cnt = 0
end_time = 0

for st, en in lst:
    if st >= end_time:
        cnt += 1
        end_time = en

print(cnt)
```

## 🔍 코드 설명
```python
lst.sort(key=lambda x: (x[1], x[0]))
```

회의를 정렬합니다.

- ``x[1]`` : 종료 시간
- ``x[0]`` : 시작 시간

즉, 종료 시간이 빠른 회의부터 정렬하고
종료 시간이 같다면 시작 시간이 빠른 회의가 먼저 오게 합니다.

```python
for st, en in lst:
    if st >= end_time:
        cnt += 1
        end_time = en
```

현재 회의의 시작 시간이

이전에 선택한 회의의 종료 시간보다 크거나 같으면

회의를 선택할 수 있습니다.

회의를 선택했다면 개수를 증가시키고,

```end_time```을 현재 회의의 종료 시간으로 갱신합니다.

## ⏱️ 시간 복잡도

정렬에 걸리는 시간은 다음과 같습니다.

``O(N log N)``

전체 회의를 한 번 순회하므로 다음과 같습니다.

``O(N)``

따라서 전체 시간 복잡도는 다음과 같습니다.

``O(N log N)``
