# 🥇 BOJ 1654 - 랜선 자르기

## 📌 문제 설명
가지고 있는 `K`개의 랜선을 잘라서  
길이가 모두 같은 `N`개의 랜선을 만들어야 합니다.

이때 만들 수 있는 **랜선의 최대 길이**를 구하는 문제입니다.

---

## 💡 풀이 아이디어

이 문제의 핵심은  
“랜선 길이를 얼마로 잘라야 `N개 이상` 만들 수 있는가?”를 찾는 것입니다.

랜선 길이가 너무 길면 만들 수 있는 개수가 부족해지고,  
랜선 길이가 짧으면 더 많은 개수를 만들 수 있습니다.

즉, **길이가 커질수록 만들 수 있는 랜선 개수는 줄어드는 단조성**이 있으므로  
이분 탐색을 사용할 수 있습니다.

---

## 알고리즘

이분 탐색

---

## ✅ 왜 이분 탐색을 쓰는가?

예를 들어 랜선 길이를 `mid`라고 정했을 때,

```python
cnt = sum(line // mid for line in lines)
```

이렇게 하면 mid 길이로 만들 수 있는 랜선 개수를 구할 수 있습니다.

- cnt >= N 이면

→ 필요한 개수 이상 만들 수 있으므로

→ 랜선 길이를 더 늘려볼 수 있습니다.
- cnt < N 이면

→ 필요한 개수를 못 만들므로

→ 랜선 길이를 줄여야 합니다.

이렇게 조건에 따라 탐색 범위를 줄여가며

가능한 최대 길이를 찾습니다.

---

## ⚙️ 핵심 코드
### 1. 이분 탐색 범위 설정
```python
start = 1
end = max(lines)
```
이유:

- mid = 0이 될 수 있음
- line // mid → 0으로 나누기 발생 ❌

---

### 개수에 따라 범위 조정
```python
if cnt >= n:
    answer = mid
    start = mid + 1
else:
    end = mid - 1
```

- cnt >= n
  - 목표 개수 이상 만들 수 있음
  - 더 긴 길이도 가능한지 확인
  - 현재 mid는 가능한 값이므로 answer에 저장
  

- cnt < n
  - 목표 개수를 못 만듦
  - 길이를 줄여야 함

---

## ⏱ 시간 복잡도

이분 탐색은 길이 범위를 반씩 줄여가므로

- 이분 탐색: O(log M)

- (M = 가장 긴 랜선 길이)
- 각 탐색마다 전체 랜선을 확인: O(K)

따라서 전체 시간 복잡도는

### **O(K log M)**

---

## 🎯 정리

이 문제는 최대 길이를 구하는 문제이고,

길이에 따라 만들 수 있는 랜선 개수가 단조롭게 변하므로 이분 탐색을 사용합니다.
