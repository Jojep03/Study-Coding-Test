# [Silver III] 수열 - 2559

[문제 링크](https://www.acmicpc.net/problem/2559)

## 문제 설명

매일 측정한 온도가 주어졌을 때, 연속적인 며칠 동안의 온도 합이 가장 큰 값을 구하는 문제이다.

전체 날짜 수 `N`과 연속해서 더할 날짜 수 `K`가 주어지고, `N`일 동안의 온도가 주어진다.
이때 연속된 `K`일의 온도 합 중 최댓값을 출력한다.

## 입력

첫째 줄에 두 정수 `N`과 `K`가 주어진다.

둘째 줄에는 매일 측정한 온도 `N`개가 공백으로 구분되어 주어진다.

## 출력

연속적인 `K`일의 온도 합 중 최댓값을 출력한다.

## 예제 입력

```text
10 2
3 -2 -4 -9 0 3 7 13 8 -3
```

## 예제 출력

```text
21
```

## 풀이 아이디어

이 문제는 연속된 `K`개의 합을 계속 구해야 하므로, 매번 `K`개를 새로 더하면 시간이 오래 걸릴 수 있다.

따라서 **슬라이딩 윈도우**를 사용한다.

처음 `K`개의 합을 구한 뒤, 오른쪽으로 한 칸씩 이동하면서

* 새로 포함되는 값을 더하고
* 범위에서 빠지는 값을 뺀다

이렇게 하면 연속된 `K`개의 합을 빠르게 갱신할 수 있다.

예를 들어 `K = 2`일 때,

```text
3 -2 -4 -9 0 3 7 13 8 -3
```

처음에는

```text
3 + (-2)
```

를 구하고, 다음 구간으로 이동할 때는

```text
기존 합 - 3 + (-4)
```

처럼 계산한다.

즉, 매번 전체를 다시 더하지 않고 하나를 빼고 하나를 더하는 방식이다.

## 코드

```python
import sys

input = sys.stdin.readline

n, k = map(int, input().split())
lst = list(map(int, input().split()))

cnt = sum(lst[:k])
res = cnt

for i in range(k, n):
    cnt += lst[i]
    cnt -= lst[i - k]

    if cnt > res:
        res = cnt

print(res)
```

## 코드 설명

```python
cnt = sum(lst[:k])
res = cnt
```

처음 `K`개의 온도 합을 구한다.
이 값이 첫 번째 연속 구간의 합이므로 `res`에도 저장한다.

```python
for i in range(k, n):
```

인덱스 `k`부터 끝까지 이동하면서 새로운 값을 하나씩 포함한다.

```python
cnt += lst[i]
cnt -= lst[i - k]
```

현재 위치의 값을 새로 더하고, 구간에서 빠지는 값을 뺀다.

```python
if cnt > res:
    res = cnt
```

현재 구간의 합이 기존 최댓값보다 크면 최댓값을 갱신한다.

## 주의할 점

온도는 음수도 나올 수 있다.
따라서 최댓값을 저장하는 `res`를 `0`으로 초기화하면 안 된다.

예를 들어 모든 온도가 음수라면 실제 최댓값도 음수일 수 있기 때문이다.

그래서 처음 `K`개의 합으로 초기화해야 한다.

```python
res = cnt
```

## 시간 복잡도

처음 `K`개의 합을 구하는 데 `O(K)`가 걸리고, 이후 전체 배열을 한 번 순회한다.

따라서 시간 복잡도는

```text
O(N)
```

이다.
