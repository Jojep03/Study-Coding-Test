# 💰 BOJ 6236 - 용돈 관리

## 📌 문제 설명

`N`일 동안 사용할 금액이 주어질 때,  
통장에서 한 번 인출할 때마다 항상 같은 금액 `K`를 인출한다고 가정한다.

총 인출 횟수를 `M`번 이하로 하면서  
모든 날의 지출을 감당할 수 있는 **최소 인출 금액 `K`**를 구하는 문제이다.

---

## 💡 풀이 아이디어

이 문제는 **이분 탐색 (Binary Search)** 을 활용하는 대표적인 문제이다.

### ✔️ 핵심 개념

- 인출 금액 `K`가 **작을수록** 인출 횟수는 **많아진다**
- 인출 금액 `K`가 **클수록** 인출 횟수는 **적어진다**

👉 즉, **단조성**이 존재하므로 이분 탐색 가능

---

## 🔍 탐색 과정

### 1. 인출 금액 범위 설정

```python
start = max(x)
end = sum(x)
```

* ``start = max(x)``

    하루 지출보다 작은 금액은 그날 사용조차 불가능하므로,

    최소 인출 금액은 하루 최대 지출 이상이어야 한다.


* ```end = sum(x)```

    전체 금액을 한 번에 인출하면 무조건 가능하므로,

    최대 범위는 전체 지출 합으로 잡는다.

### 2. 중간값(mid)을 인출 금액 후보로 설정

```python
mid = (start + end) // 2
```
mid원을 한 번 인출한다고 가정하고,

이 금액으로 며칠을 버틸 수 있는지 확인한다.


### 3. 조건에 따라 탐색 방향 결정
* **cnt <= m**

    👉 M번 이하로 가능

    👉 더 작은 금액으로도 가능한지 탐색

    👉 end = mid - 1

* **cnt > m**
  
    👉 인출 횟수가 너무 많음
  
  👉 더 큰 금액이 필요

  👉 start = mid + 1


---

## ⚡ 시간 복잡도
* 이분 탐색: **`O(log(sum(x)))`**
* 각 탐색마다 전체 지출 순회: **`O(N)`**

👉 총 시간 복잡도:
```python
O(N log(sum(x)))
```

---

## 🎯 핵심 포인트 정리
- 인출 금액 K가 커질수록 인출 횟수는 줄어든다
- 단조성이 있으므로 **이분 탐색 가능**
- ``cnt <= m``이면 가능한 금액
- 최소값을 구하는 문제이므로 **조건 만족 시 왼쪽 탐색**
- 시작 범위는 ``max(x)``부터 ``sum(x)``까지
