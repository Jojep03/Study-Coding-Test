# BOJ 1541 - 잃어버린 괄호

## 풀이 아이디어

식을 최소로 만들기 위해서는 첫 번째 `-` 이후의 모든 수를 하나의 괄호로 묶어 빼면 된다.

예를 들어

`55-50+40`

은

`55-(50+40)`

으로 계산해야 가장 작은 값이 나온다.

따라서 문자열을 `-` 기준으로 분리한 뒤,

* 첫 번째 그룹은 모두 더하고
* 나머지 그룹들은 각각 합을 구한 후 빼주면 된다.

## 구현

1. 입력 문자열을 `-` 기준으로 분리한다.
2. 첫 번째 그룹의 숫자들을 모두 더해 초기값으로 설정한다.
3. 두 번째 그룹부터는 `+` 기준으로 분리하여 합을 구한 뒤 결과에서 뺀다.


## 핵심 코드 설명

```python
res = sum(map(int, split_x[0].split("+")))
```

`-`로 나누었을 때 첫 번째 덩어리는 무조건 더해지는 부분이다.

예를 들어

```text
55-50+40
```

이라면 `split_x[0]`은 `"55"`이고, 이 값은 결과에 더해준다.

```python
for y in split_x[1:]:
    res -= sum(map(int, y.split("+")))
```

`split_x[1:]`은 첫 번째 `-` 뒤에 있는 모든 덩어리들이다.

이 문제는 첫 번째 `-` 이후의 값들을 전부 빼야 최솟값이 되므로, 각 덩어리를 `+` 기준으로 나누어 합을 구한 뒤 `res`에서 빼준다.

예를 들어

```text
55-50+40
```

은

```text
55 - (50 + 40)
```

으로 계산된다.


## 시간 복잡도

* 문자열을 한 번씩만 순회하므로 `O(N)`

## 배운 점

처음에는 `+`와 `-`를 모두 직접 처리하려고 했지만,
첫 번째 `-` 이후의 모든 값을 빼면 된다는 그리디 아이디어를 이용하면 훨씬 간단하게 해결할 수 있었다.
