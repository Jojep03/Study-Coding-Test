# [BOJ] 10025 - 게으른 백곰

## 문제

백곰이 한 위치에 서서 좌우로 `K`만큼 떨어진 범위 안의 얼음을 먹을 수 있을 때, 먹을 수 있는 얼음의 최대 양을 구하는 문제이다.

## 풀이 아이디어

얼음의 위치를 기준으로 정렬한 뒤 **투 포인터**를 사용한다.

백곰이 먹을 수 있는 범위는 한 위치를 기준으로

```text
[x - K, x + K]
```

이므로, 한 번에 포함할 수 있는 가장 왼쪽 얼음과 가장 오른쪽 얼음의 **거리 차이는**

```text
2 * K
```

이다.

따라서 현재 구간에서

```python
right_x - left_x > 2 * K
```

가 되면 범위를 벗어난 것이므로 `left`를 이동하면서 왼쪽 얼음을 제거한다.

### 진행 과정

1. `(위치, 얼음 양)` 형태로 저장한다.
2. 위치를 기준으로 오름차순 정렬한다.
3. `right`를 하나씩 이동하며 얼음 양을 `temp`에 추가한다.
4. 현재 구간의 거리 차이가 `2 * K`보다 크면 `left`를 이동하며 얼음을 제거한다.
5. 매번 현재 얼음 총합의 최댓값을 갱신한다.

## 코드

```python
import sys

input = sys.stdin.readline

n, k = map(int, input().split())

lst = []
for _ in range(n):
    g, x = map(int, input().split())
    lst.append((x, g))

lst.sort(key=lambda x: x[1])

left = 0
temp = 0
res = 0

for right in range(n):
    r_x, r_g = lst[right]
    temp += r_g

    while r_x - lst[left][0] > 2 * k:
        temp -= lst[left][1]
        left += 1

    res = max(res, temp)

print(res)
```

## 핵심

백곰이 한 위치에서 좌우로 `K`만큼 먹을 수 있으므로, 먹을 수 있는 정수 좌표의 개수는

```text
2K + 1
```

개이다.

하지만 투 포인터에서 비교하는 것은 **좌표 개수**가 아니라 **가장 왼쪽 좌표와 가장 오른쪽 좌표의 거리 차이**이다.

예를 들어 `K = 3`이라면

```text
7 8 9 10 11 12 13
```

총 `7개 = 2K + 1`의 위치를 포함할 수 있지만,

양 끝 좌표의 차이는

```text
13 - 7 = 6 = 2K
```

이다.

따라서 현재 구간에서

```python
r_x - lst[left][0] > 2 * k
```

이면 백곰이 한 번에 먹을 수 있는 범위를 벗어난 것이므로 `left`를 이동하며 범위를 줄여준다.

또한 `right`는 새로운 얼음을 하나씩 추가하고, 범위를 초과했을 때만 `left`를 이동하므로 각 원소는 최대 한 번씩 들어오고 빠진다.

즉,

```text
right → 새로운 얼음 추가
left  → 범위를 초과했을 때 제거
```

하는 방식으로 현재 구간의 얼음 합을 효율적으로 관리할 수 있다.


## 시간 복잡도

* 정렬: `O(N log N)`
* 투 포인터 탐색: `O(N)`

전체 시간 복잡도는

```text
O(N log N)
```

이다.

## 배운 점

* 좌표가 연속적으로 주어지지 않더라도 정렬 후 투 포인터로 구간을 관리할 수 있다.
* 좌우로 `K`만큼 이동할 수 있다면 전체 포함 가능한 거리 차이는 `2K`이다.
* `right`의 값을 먼저 추가하고, 범위를 초과하면 `left`를 줄이는 방식으로 슬라이딩 윈도우를 구현할 수 있다.
