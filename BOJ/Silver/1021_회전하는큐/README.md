# 🥈 BOJ 1021 - 회전하는 큐

## 📌 문제 링크
[백준 1021번 - 회전하는 큐](https://www.acmicpc.net/problem/1021)

---

## 📖 문제 설명
크기 `N`의 원형 큐에 `1`부터 `N`까지의 수가 순서대로 들어 있습니다.  
이 큐에서 주어진 수들을 순서대로 꺼내려고 할 때, 다음 3가지 연산을 사용할 수 있습니다.

1. 첫 번째 원소를 뽑아낸다.
2. 왼쪽으로 한 칸 이동한다.  
   (`[1, 2, 3] → [2, 3, 1]`)
3. 오른쪽으로 한 칸 이동한다.  
   (`[1, 2, 3] → [3, 1, 2]`)

이때, **2번과 3번 연산의 최소 횟수**를 구하는 문제입니다.

---

## 💡 풀이 아이디어
`deque`를 사용하면 양쪽 끝에서 원소를 꺼내거나 넣는 작업을 빠르게 처리할 수 있습니다.

현재 꺼내야 하는 숫자의 위치를 구한 뒤,

- **왼쪽 회전 횟수** = `target`
- **오른쪽 회전 횟수** = `len(lst) - target`

이 됩니다.

즉, 현재 목표 숫자를 맨 앞으로 가져오기 위해  
왼쪽으로 돌리는 것이 빠른지, 오른쪽으로 돌리는 것이 빠른지 비교해서  
더 적은 쪽으로 회전시키면 됩니다.

그 후 맨 앞 원소를 `popleft()`로 제거합니다.

---

## ✅ 핵심 포인트
- 회전 방향을 정할 때는 **처음 입력받은 `n`이 아니라 현재 큐 길이 `len(lst)` 기준**으로 판단해야 합니다.
- 현재 목표 숫자의 인덱스를 `lst.index(x)`로 찾습니다.
- `target <= len(lst) // 2` 이면 왼쪽 회전이 더 빠르거나 같습니다.
- 아니면 오른쪽 회전을 수행합니다.

---

## 🧠 알고리즘
1. `1`부터 `N`까지를 `deque`에 저장합니다.
2. 꺼내야 할 숫자를 하나씩 확인합니다.
3. 현재 숫자의 위치(`index`)를 찾습니다.
4. 왼쪽 회전과 오른쪽 회전 중 더 적은 쪽을 선택합니다.
5. 회전 횟수를 누적합니다.
6. 맨 앞 숫자를 제거합니다.
7. 모든 숫자에 대해 반복한 뒤 총 회전 횟수를 출력합니다.

---

## 💻 코드
```python
import sys
from collections import deque

input = sys.stdin.readline

n, m = map(int, input().split())
loc = list(map(int, input().split()))

lst = deque(range(1, n + 1))
cnt = 0
for x in loc:
    target = lst.index(x)

    if target <= len(lst) // 2:
        for _ in range(target):
            lst.append(lst.popleft())
            cnt += 1

    else:
        for _ in range(len(lst) - target):
            lst.appendleft(lst.pop())
            cnt += 1

    lst.popleft()
print(cnt)
```

## ⏱️ 시간 복잡도
- ``lst.index(x)``가 최악의 경우 ``O(N)``
- 회전 연산도 최악의 경우 ``O(N)``

따라서 전체적으로는 대략 **O(M × N)** 입니다.

문제의 제한에서는 충분히 통과할 수 있습니다.

## ✨ 정리

이 문제는 ``deque``의 회전 개념을 직접 구현하는 문제입니다.

핵심은

**현재 목표 원소를 맨 앞으로 가져올 때, 왼쪽과 오른쪽 중 어느 방향이 더 빠른지 판단하는 것입니다.**

``deque``를 활용하면 회전 과정을 직관적으로 구현할 수 있어서
시뮬레이션 문제 풀이 연습에 좋은 문제입니다.
