# 🖨️ BOJ 1966 - 프린터 큐

## 📌 문제 링크
[백준 1966번 - 프린터 큐](https://www.acmicpc.net/problem/1966)


## 📖 문제 설명

여러 개의 문서가 프린터 큐에 들어있고,  
각 문서마다 중요도가 주어집니다.

프린터는 다음 규칙으로 문서를 출력합니다.

1. 현재 큐의 맨 앞 문서를 꺼냅니다.
2. 나머지 문서 중 더 높은 중요도를 가진 문서가 하나라도 있으면,  
   이 문서를 다시 큐의 맨 뒤에 넣습니다.
3. 그렇지 않으면 바로 출력합니다.

이때, 처음 큐에서 `m`번째에 있던 문서가 몇 번째로 출력되는지 구하는 문제입니다.


## 💡 풀이 아이디어

문서의 **위치 정보**와 **중요도 정보**를 같이 관리해야 합니다.

이 풀이에서는 두 개의 `deque`를 사용했습니다.

- `lst` : 문서 번호를 저장하는 큐
- `imp` : 문서 중요도를 저장하는 큐

### 핵심 과정

1. 처음에 문서 번호를 `1 ~ n`으로 저장합니다.
2. 찾고 싶은 문서의 번호를 `target = lst[m]`으로 저장합니다.
3. 현재 중요도들 중 최댓값 `max(imp)`를 찾습니다.
4. 그 최댓값이 맨 앞에 오도록 큐를 회전합니다.
5. 맨 앞 문서를 출력합니다.
6. 그 문서가 `target`이면 출력 순서를 출력하고 종료합니다.

즉,  
매번 **현재 가장 중요한 문서가 앞에 오도록 회전한 뒤 출력**하는 방식입니다.


## 🔍 중요 코드 설명

1. 문서 번호와 중요도 분리 저장
```python
lst = deque(range(1, n + 1))
imp = deque(map(int, input().split()))

```
- lst에는 문서 번호를 저장합니다.
- imp에는 각 문서의 중요도를 저장합니다.
- 두 큐를 항상 같이 회전시켜야 문서와 중요도가 서로 맞게 유지됩니다.


2. 가장 큰 중요도가 앞에 오도록 회전
```python
for _ in range(imp.index(max_imp)):
    lst.append(lst.popleft())
    imp.append(imp.popleft())
```

- imp.index(max_imp) : 가장 큰 중요도가 처음 등장하는 위치
- 그 위치만큼 큐를 왼쪽으로 회전하면,
- 가장 큰 중요도를 가진 문서가 맨 앞에 오게 됩니다.

문서 번호 큐 lst도 같이 회전해야
문서 번호와 중요도가 계속 일치합니다.

3. 출력 후 target인지 확인
```python
if lst[0] != target:
    lst.popleft()
    imp.popleft()
    cnt += 1
else:
    cnt += 1
    break
```

- 맨 앞 문서를 출력합니다.
- 그 문서가 찾던 문서가 아니면 계속 진행합니다.
- 찾던 문서라면 몇 번째 출력인지 cnt를 세고 종료합니다.

## ⏱️ 시간 복잡도
- 문서를 하나 출력할 때마다 max(imp)와 imp.index(max_imp)를 사용합니다.
- 각각 최악의 경우 O(n)이 걸릴 수 있습니다.
- 전체 문서 수가 n개이므로 전체 시간 복잡도는 대략:

``O(n^2)``

이 문제의 입력 크기에서는 충분히 통과 가능합니다.

## ✅ 정리

이 문제의 핵심은:

- **현재 가장 높은 중요도를 가진 문서가 먼저 출력된다**
- 따라서 매번
  - 최대 중요도를 찾고
  - 그 문서가 맨 앞에 오도록 회전한 뒤
  - 출력 순서를 세면 된다

이 풀이에서는 문서 번호와 중요도를 각각 ``deque``로 관리하여

직관적으로 구현할 수 있습니다.
