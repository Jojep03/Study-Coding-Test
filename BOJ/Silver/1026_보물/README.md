# [백준 1026번] 보물 - Python

## 문제 설명

길이가 `N`인 두 배열 `A`, `B`가 주어진다.

다음 식의 값을 최소로 만들어야 한다.

```python
S = A[0] * B[0] + A[1] * B[1] + ... + A[N-1] * B[N-1]
```

배열 `A`는 재배열할 수 있지만, 문제 조건상 배열 `B`는 재배열하면 안 된다고 되어 있다.

하지만 최솟값만 구하면 되기 때문에, 가장 작은 `A`와 가장 큰 `B`를 곱하는 방식으로 계산하면 된다.

---

## 풀이 아이디어

곱의 합을 최소로 만들려면

* 작은 수는 큰 수와 곱하고
* 큰 수는 작은 수와 곱해야 한다.

즉, 매번

```python
A에서 가장 작은 값
B에서 가장 큰 값
```

을 골라 곱해주면 된다.

그 후 사용한 값은 배열에서 제거한다.

---

## 코드

```python
import sys

input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))
b = list(map(int, input().split()))

res = 0

for _ in range(n):
    min_a = min(a)
    max_b = max(b)

    res += min_a * max_b

    a.pop(a.index(min_a))
    b.pop(b.index(max_b))

print(res)
```

---

## 코드 설명

```python
min_a = min(a)
max_b = max(b)
```

현재 배열 `A`에서 가장 작은 값과 배열 `B`에서 가장 큰 값을 찾는다.

```python
res += min_a * max_b
```

두 값을 곱해서 결과값에 더한다.

```python
a.pop(a.index(min_a))
b.pop(b.index(max_b))
```

이미 사용한 값은 다시 사용하면 안 되므로 각각의 배열에서 제거한다.

이 과정을 `N`번 반복하면 모든 원소를 한 번씩 사용하게 된다.

---

## 예제

### 입력

```text
5
1 1 1 6 0
2 7 8 3 1
```

### 과정

`A`에서 작은 값부터 고르고, `B`에서 큰 값부터 고르면 다음과 같다.

```text
0 * 8 = 0
1 * 7 = 7
1 * 3 = 3
1 * 2 = 2
6 * 1 = 6
```

따라서

```text
0 + 7 + 3 + 2 + 6 = 18
```

### 출력

```text
18
```

---

## 시간 복잡도

반복문은 `N`번 실행된다.

각 반복마다 `min()`, `max()`, `index()`, `pop()`을 사용하므로 전체 시간 복잡도는

```text
O(N^2)
```

이다.

하지만 백준 1026번은 `N`의 크기가 작기 때문에 이 풀이로 충분히 통과할 수 있다.

---

## 정리

이 문제의 핵심은 다음과 같다.

```text
곱의 합을 최소로 만들려면
가장 작은 수와 가장 큰 수를 곱해야 한다.
```

따라서 `A`의 최솟값과 `B`의 최댓값을 차례대로 곱하면서 더하면 정답을 구할 수 있다.
