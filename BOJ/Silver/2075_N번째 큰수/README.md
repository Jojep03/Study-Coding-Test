# BOJ 2075 N번째 큰 수

문제 링크  
https://www.acmicpc.net/problem/2075

---

## 문제 설명

N × N 크기의 표에 숫자가 주어진다.  

이 표에 있는 **모든 숫자 중에서 N번째로 큰 수**를 구하는 문제이다.


## 알고리즘

힙 (Heap)  
최소 힙 (Min Heap)

---

## 풀이 아이디어

모든 숫자를 저장한 뒤 정렬하면 해결할 수 있지만  
N의 최대값이 **1500**이기 때문에

```
1500 × 1500 = 2,250,000
```

개의 숫자가 입력된다.

모든 값을 정렬하면 **메모리와 시간이 비효율적**이다.

따라서 **크기가 N인 최소 힙을 유지하는 방식**으로 해결한다.

핵심 아이디어

> 힙에는 항상 **큰 값 N개만 유지**한다.

### 처리 과정

1. 숫자를 하나씩 읽는다
2. 힙의 크기가 N보다 작으면 그대로 삽입
3. 힙의 크기가 N이라면  
   - 현재 숫자가 힙의 최솟값보다 큰 경우에만 교체

최소 힙의 특징 때문에

```
heap[0] = 힙에서 가장 작은 값
```

이고,

힙에는 **가장 큰 N개의 숫자만 남게 된다.**

따라서

```
heap[0] = 전체 숫자 중 N번째 큰 값
```

이 된다.

---

## 주의할 점

힙의 최솟값을 비교할 때

```
min(heap)
```

을 사용하면 시간초과가 발생할 수 있다.

이유

```
min(heap) → O(N)
heap[0] → O(1)
```

따라서 반드시

```
heap[0]
```

을 사용해야 한다.

---

## 시간복잡도

전체 숫자 개수

```
N²
```

힙 연산

```
O(log N)
```

따라서 전체 시간복잡도

```
O(N² log N)
```
