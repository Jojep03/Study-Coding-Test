# 백준 11047번: 동전 0

## 문제 설명

동전의 종류가 주어졌을 때, 주어진 금액 `K`를 만드는 데 필요한 동전 개수의 최솟값을 구하는 문제이다.

동전의 가치들은 서로 배수 관계를 이루고 있기 때문에, 큰 동전부터 최대한 많이 사용하는 그리디 방식으로 최적해를 구할 수 있다.

## 문제 풀이

가장 큰 동전부터 확인하면서 현재 금액 `K`보다 작거나 같은 동전을 최대한 많이 사용한다.

해당 동전으로 사용할 수 있는 개수는 다음과 같이 구할 수 있다.

```python
K // coin
```

동전을 사용한 뒤에는 남은 금액만 다시 계산한다.

```python

k -= coin * cnt```

이 과정을 모든 동전에 대해 반복하면 필요한 동전 개수의 최솟값을 구할 수 있다.

## 코드

```python
import sys

input = sys.stdin.readline

n, k = map(int, input().split())
coins = [int(input()) for _ in range(n)]

res = 0

for coin in reversed(coins):
    res += k // coin
    k %= coin

print(res)
```

## 코드 설명

동전의 가치를 리스트에 저장한다.

문제에서 동전은 오름차순으로 입력되므로, 큰 동전부터 사용하기 위해 `reversed(coins)`를 사용한다.

```python
for coin in reversed(coins):
```

가치가 큰 동전부터 하나씩 확인한다.

```python
cnt += k // coin
```

현재 동전으로 만들 수 있는 최대 개수를 정답에 더한다.

```python
res += cnt
k -= coin * cnt
```

현재 동전을 사용하고 남은 금액을 다시 `k`에 저장한다.

## 핵심 아이디어

동전들이 배수 관계를 이루고 있기 때문에, 큰 동전을 먼저 사용하는 선택이 항상 최적이다.
