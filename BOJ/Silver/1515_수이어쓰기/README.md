# BOJ 1515 - 수 이어 쓰기

## 문제 링크
https://www.acmicpc.net/problem/1515

## 문제 설명

숫자 `1`부터 차례대로 이어 쓴 문자열이 있다고 하자.

123456789101112131415...

입력으로 주어진 숫자 문자열 `N`이 이 문자열 안에서 **순서를 유지하면서 등장하도록 할 때**,  
마지막으로 사용한 수의 최솟값을 구하는 문제이다.

예를 들어 `1515`라면,

- `1`은 `1`에서 사용
- `5`는 `5`에서 사용
- `1`은 `10`의 `1`에서 사용
- `5`는 `15`의 `5`에서 사용

따라서 정답은 `15`이다.

---

## 알고리즘

- 구현
- 문자열
- 완전탐색

---

## 풀이 아이디어

이 문제는 복잡한 규칙을 찾는 것이 아니라  
**1부터 숫자를 하나씩 증가시키며 문자열을 순서대로 맞춰가는 시뮬레이션 문제**이다.

### 핵심 아이디어

입력 문자열 `N`에서 현재 몇 번째 문자까지 맞췄는지를 나타내는 인덱스 `idx`를 둔다.

그리고 `1, 2, 3, ...`을 하나씩 확인하면서  
각 숫자의 자릿수를 문자열로 바꿔 비교한다.

- 현재 숫자 `num`을 문자열로 변환
- 각 문자 `ch`를 순회
- `ch == N[idx]`이면 `idx += 1`
- `idx == len(N)`이면 종료

---

### 배운점

- 규칙 찾기보다 직접 시뮬레이션이 더 쉬운 문제

- 문자열은 포인터 하나로 처리 가능

- 완전탐색도 조건에 따라 충분히 효율적일 수 있음
