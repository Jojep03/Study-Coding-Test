# 🌲 BOJ 2805 - 나무 자르기

## 📌 문제 설명

나무의 높이가 주어질 때, 절단기를 이용하여 나무를 잘라 **총 길이 M 이상**을 얻으려고 한다.

절단기의 높이를 설정하면, 해당 높이보다 큰 나무는 잘리고
잘린 부분의 길이를 모두 합한 값이 된다.

👉 **M 이상을 얻을 수 있는 절단기 높이의 최댓값**을 구하는 문제

---

## 💡 풀이 아이디어

이 문제는 **이분 탐색 (Binary Search)**을 활용하는 대표적인 문제이다.

### ✔️ 핵심 개념

* 절단기 높이가 **높아질수록** 얻는 나무 길이는 **감소**
* 절단기 높이가 **낮아질수록** 얻는 나무 길이는 **증가**

👉 즉, **단조성**이 존재하므로 이분 탐색 가능

---

## 🔍 탐색 과정

1. 절단기 높이 범위 설정

```
start = 0
end = max(trees)
```

2. 중간값(mid)을 절단기 높이로 설정

```
mid = (start + end) // 2
```

3. 잘린 나무 길이 계산

```
cnt += (tree - mid)  (단, tree > mid)
```

4. 조건에 따라 탐색 방향 결정

* `cnt >= m`
  👉 충분히 나무를 얻음 → 더 높게 자를 수 있는지 탐색
  👉 `start = mid + 1`

* `cnt < m`
  👉 나무 부족 → 더 낮게 잘라야 함
  👉 `end = mid - 1`

---


## ⚡ 시간 복잡도

* 이분 탐색: `O(log H)`
* 각 탐색마다 나무 순회: `O(N)`

👉 총 시간 복잡도:

```
O(N log H)
```

---

## 🎯 핵심 포인트 정리

* 단조성이 있는 문제 → **이분 탐색**
* `cnt >= m`일 때 **answer 갱신**
* 최대값을 구하는 문제 → **조건 만족 시 오른쪽 탐색**
